# 연구노트 — 4050 여성 체형 데이터 리포트 빌드 스펙

작성일: 2026-07-20 (코워크 기획, 구현은 로컬 Claude Code 담당)

## 1. 목적

구글에게 "내몸에미소는 자체 데이터를 가진 전문 브랜드"임을 인정받기 위한 원본 리서치 콘텐츠. 방문자 유입용이 아니라 백링크·인용·E-E-A-T 신호용 — 그래서 CTA 없음, 전환 목적 아님.
(근거: 원본 데이터 콘텐츠는 의견성 글보다 백링크 6.4배, 헬스케어 SEO에서 자체 연구를 핵심 백링크 전략으로 꼽음 — 2026-07-20 대화에서 조사)

## 2. IA / 배치

새 메인 메뉴 항목 만들지 않음. 대신:

- 몸 상태 체크(`/check`) 완료 화면 — "이 데이터가 쌓여 만들어지는 연구노트 보기" 링크
- 칼럼(`/blog`) 목록 상단/사이드 — 연구노트 카드 1개
- 푸터 — "연구노트" 링크 1개 (현재 푸터엔 지도 링크·개인정보처리방침만 있어 여유 있음)

"연구노트" 허브 페이지 하나를 만들고 그 안에 리포트 카드(현재는 4050 여성 1개, 추후 청소년 등 추가)를 넣는 구조. 리포트가 늘어나도 메뉴/배치 자리는 항상 허브 1개로 고정.

## 3. 페이지 섹션 구조 (확정)

1. 도입부 — "동탄 4050 여성 체형 데이터 리포트" + 표본 수 자동 표시 + "초기 관찰 기록입니다" 명시(과장 금지)
2. 데이터 출처/방법론 — 바디닷 3D 체형측정, 2026년 6월부터 축적, 익명화(이름 제외, 연령대+성별+수치만), "내몸에미소 방문 회원 기준 관찰 데이터"로 정직하게 프레이밍(전체 통계 아님)
3. 핵심 발견(시각화) — 관찰 평균 각도 + 움직임평가 점수. 항목별 표본수(N)가 다르므로 반드시 N 병기. 정상범위 비교는 바디닷 공식 기준표 확보 전까지 넣지 않음(임의 기준 절대 금지)
4. 실제 사례 연결 — 관련 임상노트 내부링크(SEO 겸함)
5. 왜 중요한지 — 4050 여성 체형문제 맥락 설명
6. CTA 없음(고정)
7. 업데이트 표시 — "표본 OO명 · 마지막 업데이트 날짜 · 다음 업데이트 예정"

## 5. E-E-A-T / 구글 신뢰신호 요구사항

- JSON-LD 구조화 데이터: author는 실명(Person) 유지 가능 — 화면엔 브랜드만 노출, 백엔드 메타데이터만 실명(feedback_brand_over_personal_branding 원칙과 상충 안 됨)
- dateModified를 실제 갱신 시점으로 정확히 반영(분기 업데이트마다 갱신)
- 방법론 섹션(2번) 문구는 절대 생략하지 말 것 — 표본 크기·수집기간·한계를 명시하는 것 자체가 신뢰 신호
- 정상범위 등 임의 기준값 텍스트에 절대 넣지 말 것(무근거 수치는 오히려 신뢰도 훼손)
