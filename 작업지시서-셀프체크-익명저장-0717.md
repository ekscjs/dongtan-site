# [작업지시서] 셀프체크 답변 익명 저장 — 연구노트 데이터 축적 파이프

**작업 폴더:** dongtan-site (bodymiso.com)
**작성:** 2026-07-16 (내일 작업용)

---

## ■ 목표

셀프체크(`/check`) 완료 시 답변을 **익명으로** Supabase에 저장하기 시작한다.
(나중에 연구노트 집계·또래비교·경과추적의 원료가 됨)

## ■ 현재 상태 (확인됨 — 다시 조사하지 말 것)

- `src/app/check/CheckQuiz.tsx` 는 결과를 ①localStorage ②`/api/track` 집계이벤트(`self_check_complete`: `{type, risk 라벨}`만) 에만 남긴다.
- 실제 답변(8문항)은 서버에 저장되지 않아 매번 유실되고 있다. 이걸 고친다.

## ■ 원칙 (확장형 뼈대 / 최소 구현)

- 나중에 계정·경과추적·벤치마킹이 "붙기"만 하면 되게 구조화 + 익명ID로 저장.
- 지금은 "저장 파이프"만. 로그인·계정·벤치마킹 화면·URL 변경은 하지 말 것.
- 개인정보(이름·연락처) 절대 받지 않는다. 순수 익명. (연락처는 기존 `leads` 테이블 담당)

## ■ 할 일

**1) Supabase 테이블 생성**

`supabase_self_check.sql` 새로 만들고 `supabase_leads.sql` 패턴 그대로: RLS on, 공개정책 없음, 서버 service role만 접근.

컬럼:
- `id` bigint identity PK / `created_at` timestamptz default now()
- `anon_id` text ← 기존 트래킹의 `visitor_id` 재사용 (신규 UUID 만들지 말 것)
- `sitting_hours` text, `main_area` text, `worse_when` text, `forward_head` text, `round_shoulder` text, `uneven_stance` text, `temporary_relief` text, `pain_severity` text ← `src/app/check/data.ts` 8문항 순서대로 "의미 있는 필드명"으로 저장
- `answers_raw` jsonb ← 원본 답변 배열도 함께 (스키마 바뀌어도 복구 가능)
- `result_type` text, `risk_label` text, `risk_score` int
- `is_retest` boolean default false

**2) `/api/self-check` POST 라우트 신설**

- `@/lib/supabaseAdmin` 의 `supabaseAdmin` 으로 insert.
- `/api/track/route.ts` 구조를 그대로 참고.

**3) `CheckQuiz.tsx` 에 저장 호출 추가**

- 최초 완료 지점: 현재 `track("self_check_complete")` 치는 자리(약 line 157)
- 재진단 지점: `track("self_check_retest")` 자리(약 line 145) → `is_retest=true` 로 저장
- `answers` 상태배열 + `data.ts` questions 를 매핑해 각 필드를 채운다.

**4) anon_id**

- 방문자 트래킹이 이미 쓰는 localStorage `visitor_id` 를 그대로 사용. (없을 때만 생성)

## ■ 검증

- 로컬 dev(`npm run dev`)에서 퀴즈 완료 / 재진단 각각 테스트 → `self_check_results` 에 익명 row 들어오는지 확인.
- 라이브(bodymiso.com)에서 반복 테스트 금지 (애널리틱스 오염됨). 배포 확인만 최소로.
- `npm run build` 통과 후 커밋.
