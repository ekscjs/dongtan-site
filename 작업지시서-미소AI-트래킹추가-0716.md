# 작업지시서 — 미소AI 사용현황 트래킹 추가 (2026-07-16)

> 배경: 미소AI(`/ask`) 관련 클릭·질문제출이 지금 애널리틱스에 전혀 안 잡힘. 기존 사이트에 이미
> 있는 이벤트 트래킹 패턴(`track()` 함수 → `/api/track` → `events` 테이블, admin/analytics에서
> `events.counts["이벤트이름"]`으로 집계)을 미소AI에도 똑같이 붙인다. 백엔드(`/api/admin/events`)는
> 이미 모든 `event_name`을 자동 집계하므로 **수정 불필요** — 프론트 트래킹 호출 추가 + 대시보드에
> 카드 섹션 하나만 추가하면 됨.
>
> 참고: 실제 질문 문구 자체는 이미 `ask_logs` 테이블에 다 쌓이고 있음(어제 만듦) — 이건 그것과
> 별개로 "클릭·제출 횟수"를 대시보드에서 바로 보기 위한 가벼운 카운트 트래킹임.

## 1. `src/app/check/CheckQuiz.tsx` — 카드 클릭 트래킹

"미소AI" 카드의 `<Link href="/ask" ...>` 태그에 `onClick` 추가 (다른 카드들 트래킹 패턴과 동일):

```tsx
<Link
  href="/ask"
  onClick={() => track("ai_concierge_click", { from: "check" })}
  className="block bg-white rounded-2xl p-6 shadow-sm border border-gray-100 hover:border-[#7B2D8B] transition-colors group"
>
```
(이 파일엔 이미 `track()` 함수가 정의돼 있음 — 그대로 재사용, 새로 만들 필요 없음)

## 2. `src/app/ask/AskConcierge.tsx` — 질문제출·클릭 트래킹

### 2-1. `track()` 함수 추가
파일 상단, `EXAMPLES` 상수 선언 근처(또는 아무 최상위 위치)에 `CheckQuiz.tsx`에 있는 것과 똑같은
함수 추가:

```tsx
function track(event: string, properties?: Record<string, unknown>) {
  fetch("/api/track", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ event, properties: properties ?? {} }),
  }).catch(() => {});
}
```

### 2-2. 질문 제출 결과 트래킹
`submit()` 함수 안 `setResult(data as AskResult);` 줄을 아래로 교체:

```tsx
      const r = data as AskResult;
      setResult(r);
      track("ai_ask_submit", {
        result: r.is_emergency ? "emergency" : r.limited ? "limited" : "normal",
      });
```

### 2-3. 관련 글 클릭 트래킹
`related_posts.map` 안의 `<Link key={p.slug} href={...} className=...>` 에 `onClick` 추가:

```tsx
<Link
  key={p.slug}
  href={`/blog/${p.slug}`}
  onClick={() => track("ai_ask_click_post", { slug: p.slug })}
  className="block bg-white rounded-xl border border-gray-100 px-4 py-3.5 hover:border-[#7B2D8B] hover:bg-[#FAF5FB] transition-colors"
>
```

### 2-4. 방(tools) 클릭 트래킹
`tools.map` 안의 `<Link key={t.href} href={t.href} className=...>` 에 `onClick` 추가:

```tsx
<Link
  key={t.href}
  href={t.href}
  onClick={() => track("ai_ask_click_tool", { href: t.href })}
  className="flex items-center gap-3 bg-white rounded-xl border border-gray-100 px-4 py-3.5 hover:border-[#7B2D8B] hover:bg-[#FAF5FB] transition-colors group"
>
```

(상담 CTA `KakaoButton`은 이미 자체적으로 `kakao_click` 이벤트를 페이지 경로(`/ask`)와 함께
자동으로 기록하고 있어서 — **손댈 필요 없음**.)

## 3. `src/app/admin/analytics/page.tsx` — 대시보드에 "미소AI 사용 현황" 카드 추가

L698 근처, "몸 상태 체크 사용 현황" 카드(`{/* ── 이벤트 통계 ── */}`로 시작하는 `bg-white
rounded-2xl shadow p-6 mb-6` 블록)가 끝나는 `</div>` 바로 다음, "구글 검색어 순위표" 섹션
시작 전에 아래 블록을 새로 추가:

```tsx
        <div className="bg-white rounded-2xl shadow p-6 mb-6">
          <h2 className="text-sm font-semibold text-gray-700 mb-4">
            미소AI 사용 현황 <span className="text-xs font-normal text-gray-400">(최근 30일)</span>
          </h2>
          {!events ? (
            <p className="text-xs text-gray-400">불러오는 중...</p>
          ) : (
            <div className="grid grid-cols-2 md:grid-cols-4 gap-4">
              <div className="bg-[#FAF5FB] rounded-xl p-4 text-center">
                <p className="text-xs text-gray-500 mb-1">카드 클릭</p>
                <p className="text-3xl font-bold text-[#7B2D8B]">{events.counts["ai_concierge_click"] ?? 0}</p>
                <p className="text-xs text-gray-400 mt-1">회</p>
              </div>
              <div className="bg-[#FAF5FB] rounded-xl p-4 text-center">
                <p className="text-xs text-gray-500 mb-1">질문 제출</p>
                <p className="text-3xl font-bold text-[#7B2D8B]">{events.counts["ai_ask_submit"] ?? 0}</p>
                <p className="text-xs text-gray-400 mt-1">회</p>
              </div>
              <div className="bg-[#FAF5FB] rounded-xl p-4 text-center">
                <p className="text-xs text-gray-500 mb-1">관련 글 클릭</p>
                <p className="text-3xl font-bold text-[#7B2D8B]">{events.counts["ai_ask_click_post"] ?? 0}</p>
                <p className="text-xs text-gray-400 mt-1">회</p>
              </div>
              <div className="bg-[#FAF5FB] rounded-xl p-4 text-center">
                <p className="text-xs text-gray-500 mb-1">방(자가진단·운동 등) 클릭</p>
                <p className="text-3xl font-bold text-[#7B2D8B]">{events.counts["ai_ask_click_tool"] ?? 0}</p>
                <p className="text-xs text-gray-400 mt-1">회</p>
              </div>
            </div>
          )}
        </div>
```

이 섹션은 지금 당장은 전부 0으로 보일 거고, 그건 정상임 — 사이트 론칭 한 달밖에 안 됐고 트래킹도
방금 붙인 거라 데이터가 쌓이려면 시간이 걸림. 미리 만들어두는 이유는 나중에 볼 준비를 지금
끝내두기 위함(그때 가서 또 개발할 필요 없게).

## 검증
1. 로컬 dev 서버에서 `/check` → "미소AI" 카드 클릭 → `/ask`로 이동하는지(기존과 동일하게 동작).
2. `/ask`에서 질문 하나 제출 → 정상적으로 답 나오는지(기존 기능 안 깨졌는지).
3. `npm run build` 통과.
4. **실제 트래킹 확인은 배포 후 관장님이 직접 사이트에서 미소AI 한 번 써보고, admin 대시보드
   `/admin/analytics`에서 "미소AI 사용 현황" 숫자가 1 이상으로 올라가는지 확인** — 이건 로컬
   Claude Code가 아니라 관장님이 배포 후에 확인해야 하는 부분.
