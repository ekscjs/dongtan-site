# 작업 지시서 — "더보기" 버튼 숫자 제거 + /check 페이지 줄바꿈 수정

## 1. 블로그 "더보기" 버튼에서 잔여 개수 숫자 제거

파일: `src/app/blog/page.tsx` (약 144번째 줄)

현재:
```tsx
더보기 ({filtered.length - visible}개 남음)
```

수정:
```tsx
더보기
```

이유: 글이 쌓일수록 "더보기 (90개 남음)" 같은 어색한 큰 숫자가 노출됨. 대부분 블로그가 그냥 "더보기"만 씀. (참고: 글이 100개 넘어가면 그때 숫자 페이지네이션(1 2 3 ... 다음)으로 전환하는 걸 별도로 논의하기로 함 — 이번 작업 범위 아님)

## 2. /check 페이지 카드 문구 줄바꿈 수정

파일: `src/app/check/CheckQuiz.tsx`

### 2-1. 체형 유형 진단 카드 (약 217~220번째 줄)

현재:
```tsx
<p className="text-sm text-gray-500">
  거북목 / 골반 / 허리 / 전신 — 내 몸 유형을 파악하고<br />
  7일 교정 루틴을 받아보세요
</p>
```

수정:
```tsx
<p className="text-sm text-gray-500">
  거북목 / 골반 / 허리 / 전신<br />
  내 몸 유형을 파악하고 7일 교정 루틴을 받아보세요
</p>
```

### 2-2. 통증 부위 체크 카드 (약 238~241번째 줄)

현재:
```tsx
<p className="text-sm text-gray-500">
  목·어깨·허리·골반·무릎·발목 중<br />
  아픈 부위를 클릭하면 원인 분석을 해드려요
</p>
```

수정:
```tsx
<p className="text-sm text-gray-500">
  목·어깨·허리·골반·무릎·발목<br />
  아픈 곳을 클릭하면 원인 분석을 해드려요
</p>
```

이유: 기존 문구는 `<br/>` 지점 앞에서 화면 폭이 좁아 한 번 더 자연 줄바꿈이 일어나면서 "내"나 "중" 같은 한 글자가 줄 끝에 혼자 남는 현상(고아 문자) 발생. 문구를 짧게 나눠 애초에 어중간한 곳에서 안 꺾이도록 수정.

## 3. 통증지도 결과 CTA 문구 줄바꿈 수정 (모바일 실제 스크린샷 확인됨)

파일: `src/app/check/pain/PainMap.tsx` (약 365번째 줄)

현재:
```tsx
<p className="text-purple-200 text-sm mb-5">
  센터에서는 이 결과를 바탕으로 실제 움직임을 확인하고<br />
  맞춤 운동 방향을 안내해 드립니다.
</p>
```

수정 (강제 줄바꿈 제거, 자연스럽게 흐르게):
```tsx
<p className="text-purple-200 text-sm mb-5">
  센터에서는 이 결과를 바탕으로 실제 움직임을 확인하고 맞춤 운동 방향을 안내해 드립니다.
</p>
```

이유: 모바일에서 실제로 "확인하고"가 혼자 한 줄에 남는 것 확인됨(관장 스크린샷 확인, 2026-07-14). 이 부모 div가 `text-center`라 억지로 두 줄 나눌 필요 없이 자연스럽게 흐르게 두면 알아서 정리됨. 바로 위 "AI 분석 결과, 정확한 평가는<br/>움직임 검사까지 해야 합니다." 줄은 정상 작동 중이라 그대로 둘 것.

(참고: `about/page.tsx` 104~106번 줄 원장 인용구는 실제 확인 결과 "...자격이 / 있습니다." 처럼 문장이 끝나는 지점에서 자연스럽게 갈려서 어색하지 않음 — 수정 불필요, 그대로 둘 것)
