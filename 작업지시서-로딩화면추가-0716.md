# 작업지시서 — 칼럼·몸상태체크 로딩 화면 추가 (2026-07-16, UX 버그)

> 진단 완료: ①`/blog`는 글 목록을 마운트 후 client fetch로 불러오는데, 로딩 중엔 "불러오는
> 중..." 텍스트 한 줄뿐이라 콘텐츠 높이가 짧아짐 → 어두운 배경 Footer(`bg-gray-900`)가 순간
> 위로 올라왔다가 글 목록이 로드되면 다시 아래로 밀려남 → 이게 "깜빡임/검은 배경 플래시"로
> 보임. ②`/blog`, `/check` 둘 다 네비게이션 클릭 시 Next.js가 보여줄 로딩 화면(`loading.tsx`)이
> 없어서, 페이지 전환 중 아무 피드백 없이 멈춰있다가 훅 넘어감(다른 페이지는 이 파일 구조가
> 아니라 증상 없음). `/check`는 이미 `min-h-screen`이 있어서 Footer 튀는 문제는 없음 — 로딩
> 피드백만 추가하면 됨.

## 검증
1. 로컬 dev 서버에서 다른 페이지 → "칼럼" 클릭: 전환 중 회색 스켈레톤이 바로 보이는지, 글
   목록 로드된 후 Footer가 갑자기 튀어오르지 않는지(깜빡임 사라졌는지) 확인.
2. 다른 페이지 → "몸 상태 체크" 클릭: 전환 중 스켈레톤이 바로 보이는지 확인.
3. `/blog` 게시글 상세 페이지(`/blog/[slug]`)는 건드리지 않았으니 기존처럼 정상 동작하는지만
   간단히 확인.
4. `npm run build` 통과.
