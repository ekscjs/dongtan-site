# 작업지시서 — 관리자 로그인 시도 횟수 제한 (2026-07-16, 보안)

> 배경: `/api/admin/login`에 로그인 시도 횟수 제한이 없어서 봇이 비밀번호를 무한정 계속 시도할
> 수 있는 상태. `ask_logs`에 썼던 것과 동일한 방식(IP 해시 + 테이블 카운트)으로 막는다.
> 테이블 `public.admin_login_attempts`는 이미 Supabase에 만들어져 있음(id, created_at, ip_hash,
> success — RLS 켜져 있고 공개 정책 없음, 서버만 접근 가능). **SQL은 이미 실행 완료, 코드만
> 추가하면 됨.**

## `src/app/api/admin/login/route.ts` 수정

파일 맨 위 import에 `NextRequest`, `createHash` 추가하고, `POST` 함수 시작 부분에 IP 체크 로직을
넣는다. 아래는 전체 파일의 최종 형태 — 기존 로직(비밀번호 검증·쿠키 설정) 그대로 유지하면서
앞뒤로만 추가된 것:

```ts
import { NextRequest, NextResponse } from "next/server";
import { cookies } from "next/headers";
import { createHash } from "crypto";
import bcrypt from "bcryptjs";
import { supabaseAdmin } from "@/lib/supabaseAdmin";

const MAX_ATTEMPTS = 5;
const WINDOW_MINUTES = 15;

function ipHashOf(req: NextRequest): string {
  const ip = req.headers.get("x-forwarded-for")?.split(",")[0]?.trim()
    || req.headers.get("x-real-ip")
    || "unknown";
  return createHash("sha256").update(ip + "naemiso-admin-login-salt").digest("hex");
}

export async function POST(req: NextRequest) {
  const ipHash = ipHashOf(req);
  const since = new Date(Date.now() - WINDOW_MINUTES * 60 * 1000).toISOString();

  const { count: failCount } = await supabaseAdmin
    .from("admin_login_attempts")
    .select("id", { count: "exact", head: true })
    .eq("ip_hash", ipHash)
    .eq("success", false)
    .gte("created_at", since);

  if ((failCount ?? 0) >= MAX_ATTEMPTS) {
    return NextResponse.json(
      { error: `로그인 시도가 너무 많았어요. ${WINDOW_MINUTES}분 후 다시 시도해주세요.` },
      { status: 429 }
    );
  }

  const { password } = await req.json();
  if (!password) {
    return NextResponse.json({ error: "비밀번호를 입력해주세요." }, { status: 400 });
  }

  const { data: settings } = await supabaseAdmin
    .from("admin_settings")
    .select("id, password_hash")
    .order("id", { ascending: true })
    .limit(1)
    .maybeSingle();

  let valid = false;

  if (settings?.password_hash) {
    valid = await bcrypt.compare(password, settings.password_hash);
  } else {
    // 마이그레이션 기간 폴백 — DB에 해시가 아직 없으면 기존 env var로 검증
    valid = password === process.env.ADMIN_PASSWORD;
    if (valid) {
      try {
        const hash = await bcrypt.hash(password, 10);
        const { error } = await supabaseAdmin.from("admin_settings").insert({ password_hash: hash });
        if (error) console.warn("admin_settings 자동 시드 실패:", error.message);
      } catch (e) {
        console.warn("admin_settings 자동 시드 오류:", e);
      }
    }
  }

  await supabaseAdmin.from("admin_login_attempts").insert({ ip_hash: ipHash, success: valid });

  if (!valid) {
    return NextResponse.json({ error: "비밀번호가 틀렸습니다." }, { status: 401 });
  }

  const cookieStore = await cookies();
  cookieStore.set("admin_auth", "1", {
    httpOnly: true,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    maxAge: 60 * 60 * 24 * 30,
    path: "/",
  });
  cookieStore.set("admin_flag", "1", {
    httpOnly: false,
    secure: process.env.NODE_ENV === "production",
    sameSite: "lax",
    maxAge: 60 * 60 * 24 * 30,
    path: "/",
  });

  return NextResponse.json({ ok: true });
}
```

이 파일 하나만 통째로 위 내용으로 교체하면 됨(기존 쿠키 설정 부분도 그대로 옮겨져 있음 — 값 다시
확인만 할 것). 다른 파일은 안 건드림.

## 검증
1. 일부러 틀린 비밀번호로 6번 연속 로그인 시도 → 6번째부터 429 에러("로그인 시도가 너무
   많았어요. 15분 후 다시 시도해주세요.")로 막히는지.
2. 정상 비밀번호로 로그인 시 기존처럼 잘 되는지(제한에 걸리지 않은 상태에서 테스트).
3. `npm run build` 통과.
